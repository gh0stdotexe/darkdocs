<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>Support for IPv6-only networks</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/dns64)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-is-dns64&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-is-dns-64&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What is DNS64</div><Link aria-label=&quot;Skip link to What is DNS64&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-is-dns64&quot;></Link>

<p>DNS64 is a feature allows IPv6-only networks to access IPv4 networks. DNS64 synthesizes AAAA records from A records, enabling an IPv6-only network to understand and connect to IPv4 addresses through an NAT64 gateway.</p>
<p>Default &quot;well known&quot; prefix is used: <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`64:ff9b::/96`</p>
> ## <span class=&quot;callout-icon&quot;>ðŸš§</span><p>Don&#x27;t turn this on unless you need it</p>

<p>This feature is for IPv6-only networks. If you&#x27;re on a dual-stack network (have both IPv4 and IPv6) do not enable this feature, or you will have a bad time.</p>
