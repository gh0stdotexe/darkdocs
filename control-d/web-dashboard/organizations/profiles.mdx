<header id="content-head"><div class="row clearfix"><div class="col-xs-9"># Profiles

<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>A collection of rules, sometimes known as a configuration or policy.</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/profiles)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="> ## <span class=&quot;callout-icon&quot;>ðŸ“˜</span><p>TLDR</p>

<p>Profiles are policies, or rather collections of rules and behaviors you want to enforce on a set of physical devices. Profiles can be chained in order to reduce duplication and form complex behaviors.</p>

# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-are-profiles&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-are-profiles&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What are Profiles?</div><Link aria-label=&quot;Skip link to What are Profiles?&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-are-profiles&quot;></Link>

<p>A profile is simply a collection of rules and settings you wish to enforce on a specific resolver. This can sometimes be known as a policy or configuration. A profile has the following elements that can be tweaked:</p>

- **<Link href=&quot;/docs/filters&quot; target=&quot;&quot; title=&quot;&quot;>Filters</Link>** - categories you wish to be blocked

- **<Link href=&quot;/docs/services&quot; target=&quot;&quot; title=&quot;&quot;>Services</Link>** - specific websites and services you wish to block or redirect 

- **<Link href=&quot;/docs/custom-rules&quot; target=&quot;&quot; title=&quot;&quot;>Custom Rules</Link>** - granular domain level rules 

- **Profile Options** - customizable behaviors within a Profile

# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;how-to-use-profiles&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-how-to-use-profiles&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>How to Use Profiles</div><Link aria-label=&quot;Skip link to How to Use Profiles&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#how-to-use-profiles&quot;></Link>

<p>A Profile by itself doesn&#x27;t do anything, it must be enforced on a Device in order to be used. A single profile can be enforced on one or more Devices (resolvers). </p>
