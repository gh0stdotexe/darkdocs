> ## <span class=&quot;callout-icon&quot;>ðŸš§</span><p>Captive Portals</p>

<p>If you use a lot of public Wi-Fi hotspots which have captive portals, you may find that DNS is broken on such networks, and you can&#x27;t even access the captive portal itself.<br/>
If this is the case, select **Advanced Settings** in the above menu, and toggle on the &quot;Bypass common domains&quot; option. This will exclude common captive portals from using Control D. You can also add your own domains and WiFI networks using the same menu.</p>

## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;advanced-settings&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-advanced-settings&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Advanced Settings</div><Link aria-label=&quot;Skip link to Advanced Settings&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#advanced-settings&quot;></Link>

<p>You can further customize the DNS profile by clicking on the **Advanced Settings** button. </p>
<p><span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/29af48b-image.png&quot; align=&quot;&quot; caption=&quot;&quot; height=&quot;auto&quot; title=&quot;&quot; width=&quot;auto&quot;/></span></span></p>
<p>You can adjust the following:</p>

- **Client ID** - Enter any name that you want, which will allow you to see Analytics for individual physical gadgets that all use the same Control D Device. This is part of the <Link href=&quot;/docs/device-clients&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;device-clients&quot;>Device Clients</Link> feature. 

- **Exclude Wi-FI Networks** - Here you can list Wi-Fi SSIDs where you don&#x27;t want to use Control D. Handy for work networks that may block it, or your home where you already have Control D on your router. 

- **Exclude Domains** - Here you can enter specific domain names that you wish to be excluded from Control D. These will resolve using your regular ISP assigned DNS resolver. This is handy for accessing private domains or captive WiFI portals. 
<ul>
<li>Choose Folder - You can keep a list of domains to exclude in a folder. Choose the folder and it will auto-populate the list of domains you wish to exclude, so you don&#x27;t have to type them out every time. 

</li>
<li>**Exclude Common domains** - This toggle will exclude common captive portals (such as on planes) to solve the <Link href=&quot;/docs/doesnt-work-on-captive-portals&quot; target=&quot;&quot; title=&quot;&quot;>captive portal issue</Link>.</li>
<li>**Prevent Disablement (Org Accounts Only)** - If you&#x27;re using managed (supervised mode) devices, you can toggle this option on, which will prevent the end user from disabling this DNS profile. </li>
<li>**Trust Control D Root CA **- Will bake in a Root CA certificate which can be used to display <Link href=&quot;/docs/blocked-query-response&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;blocked-query-response&quot;>Custom Block Pages</Link>.</li>
</ul>
