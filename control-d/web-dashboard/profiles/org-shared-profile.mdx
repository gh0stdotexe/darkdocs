<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>You can share your main organization Profiles with Sub-Organizations to avoid duplication.</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/shared-profile)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="> ## <span class=&quot;callout-icon&quot;>ðŸ“˜</span><p>Organizations Only</p>

<p>A business account is required to use this feature.</p>

# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-is-shared-profile&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-is-shared-profile&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What is Shared Profile</div><Link aria-label=&quot;Skip link to What is Shared Profile&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-is-shared-profile&quot;></Link>

<p>This feature only exists inside the Main Organization. Any profile owned by it can be shared by toggling the option in Profile Options section. Once you do this, it will show up inside all your Sub-Organizations, allowing you to enforce it directly on Devices owned by a sub-organizations. </p>

## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;why-use-this&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-why-use-this&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Why use this</div><Link aria-label=&quot;Skip link to Why use this&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#why-use-this&quot;></Link>

<p>You can use Sub-Organizations however you&#x27;d like, which can be just as a means of grouping Devices together. When creating Devices inside a sub-organization, you have to enforce a Profile on them. If you don&#x27;t share the main organization profiles, you will have to make new Profiles inside each sub-organization, which may force you to duplicate them. </p>
<p>By sharing the main org&#x27;s Profiles, you can avoid this duplication, and re-use the existing profiles you already have. </p>
<p>Shared Profiles will be denoted with a blue badge (as shown above) in the drop down menu when you create a Device. **These Profiles will not show up in the Profiles section of the sub-organization, and cannot be edited by an admin of a sub-organization. **</p>" style="margin-left:31px" class="rm-Markdown markdown-body rm-Markdown markdown-body ng-non-bindable" data-testid="RDMD">> ## <span class="callout-icon">ðŸ“˜</span><p>Organizations Only</p>

<p>A business account is required to use this feature.</p>
