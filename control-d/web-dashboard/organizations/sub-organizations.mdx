<header id="content-head"><div class="row clearfix"><div class="col-xs-9"># Sub Organizations

<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>Create sub-organizations in order to compartmentalize your Profiles and Devices into logical groups.</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/org-sub-organizations)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-are-sub-organizations&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-are-sub-organizations&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What are Sub-Organizations?</div><Link aria-label=&quot;Skip link to What are Sub-Organizations?&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-are-sub-organizations&quot;></Link>

<p>Sub-Organizations are simply groups of Profiles and Devices that belong together. Sub-Organizations can be any of the following:</p>

- Departments

- Physical sites 

- Customer companies (if you&#x27;re an MSP)

- Anything you wish

<p>All Organizations have the ability to create as many Sub-Organizations as needed.</p>
# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;create-sub-organization&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-create-sub-organization&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Create Sub-Organization</div><Link aria-label=&quot;Skip link to Create Sub-Organization&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#create-sub-organization&quot;></Link>

<p>To create a Sub-Organization, expand your primary Organization, select **Sub-Organizations** and click the big <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`+` button. You will be presented with the following form. </p>
<span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/19a63b1-sub-org-create.PNG&quot; title=&quot;&quot; align=&quot;center&quot; class=&quot;&quot; caption=&quot;&quot; height=&quot;auto&quot; width=&quot;auto&quot;/></span></span>

- **Name** - Name of the sub-organization

- **Contact Name and Email** - Who is responsible for this sub-organization

- **Address** - Optional physical address if any

- **Website** - Optional website if any

- **Two-Factor Authentication** - If enabled, all members are forced to enable 2FA

- **Users** - Optional cap of User type devices that can be created within the sub-org

- **Routers** - Optional cap of Router type devices that can be created within the sub-org

- **Data Storage Region** - Where Analytics data is hosted

- **<Link href=&quot;/docs/global-profile&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;global-profile&quot;>Global Profile</Link>** - Choose any parent organization&#x27;s Profile to enforce on all sub-organization&#x27;s devices.

<p>Click the **Create** button and you&#x27;re done. </p>
# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;manage-organizations&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-manage-organizations&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Manage Organizations</div><Link aria-label=&quot;Skip link to Manage Organizations&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#manage-organizations&quot;></Link>

<p>Once you have more than zero Sub-Organizations, you will see them in a table. </p>
<span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/d620f80-sub-orgs-view.PNG&quot; title=&quot;&quot; align=&quot;center&quot; class=&quot;&quot; caption=&quot;&quot; height=&quot;auto&quot; width=&quot;auto&quot;/></span></span>
<p>From this section, you can do the following:</p>

- View your Sub-Organizations and how many Users, Routers, Profiles and Admins exist in each one

- See the associated cost with the maximum amount of Users and Routers you have committed to the Sub-Organization

- From the drop down menu, you can edit Sub-Organization details, membership and delete the Sub-Organization  

- Impersonate an administrator of a Sub-Organization 

### <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;sub-organization-membership&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-sub-organization-membership&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Sub-Organization Membership</div><Link aria-label=&quot;Skip link to Sub-Organization Membership&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#sub-organization-membership&quot;></Link>

<p>By default, a Sub-Organization has no direct members (admins). Only members of the primary Organization have access to it, with the same permission level as in the primary Organization. Simply click the icon on the right side of the row in order to view the Sub-Organization as if you&#x27;re a member of it - this is called &#x27;impersonation&#x27;. This allows for creation of Profiles and Devices within the Sub-Organization. </p>
<p><span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/5e64885-image.png&quot; align=&quot;&quot; caption=&quot;&quot; height=&quot;auto&quot; title=&quot;&quot; width=&quot;auto&quot;/></span></span></p>
<p>If you wish to delegate access to a single Sub-Organization, you can select the **Add/Edit Members** option from the above drop down menu and invite folks directly into the Sub-Organization. The <Link href=&quot;/docs/org-members-permissions&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;org-members-permissions&quot;>permission scheme</Link> is the same as for the primary Organization, except invited emails would have access solely to the invited Sub-Organization. </p>
<p><span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/e29955a-image.png&quot; align=&quot;&quot; caption=&quot;&quot; height=&quot;auto&quot; title=&quot;&quot; width=&quot;auto&quot;/></span></span></p>
### <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;user-and-router-limits&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-user-and-router-limits&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>User and Router Limits</div><Link aria-label=&quot;Skip link to User and Router Limits&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#user-and-router-limits&quot;></Link>

<p>When creating a Sub-Organization, you may choose the maximum amount of User and Router devices that can exist within the Sub-Organization. For example, if you&#x27;re creating a Sub-Organization for a managed company that has 100 employees and 2 routers that you wish to secure, you should select those values when creating the Sub-Organization. You can then <Link href=&quot;/docs/provisioning&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;provisioning&quot;>provision</Link>  those Users and Devices yourself, or invite someone from the Sub-Organization to do it on your behalf. They will be limited to the number of Users and Routers you have chosen. These limits can be adjusted by an **Owner** of the primary Organization at any time.</p>
<p>**If no limit is chosen, then the sub-organization will be subject to the parent organization&#x27;s limits, and you will be able to overcommit your device counts. While overcommitted, you will not be able to create new sub-organizations or new devices in parent organization until you increase your <Link href=&quot;/docs/org-billing-products&quot; target=&quot;&quot; title=&quot;&quot; class=&quot;doc-link&quot; data-sidebar=&quot;org-billing-products&quot;>Products and Billing</Link>limits. **</p>" style="margin-left:31px" class="rm-Markdown markdown-body rm-Markdown markdown-body ng-non-bindable" data-testid="RDMD"># <div class="heading-anchor anchor waypoint" id="what-are-sub-organizations"></div><div class="heading-text"><div id="section-what-are-sub-organizations" class="heading-anchor_backwardsCompatibility"></div>What are Sub-Organizations?</div>[](#what-are-sub-organizations)
