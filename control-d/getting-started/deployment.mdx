# Deployment

This section provides a very brief overview of how to deploy and configure Control D.

## Create an Endpoint (Resolver for your Devices)

<Steps>
1. Go to the [Control D Dashboard](https://controld.com/dashboard)
2. Click on **Devices** in the left sidebar
3. Click the **+** button to create a new Device
4. Choose your platform
5. Follow the instructions for your chosen platform
</Steps>

## End User Self-Provisioning

Control D can be deployed in several ways, depending on your needs and preferences.

### Software is Optional

The simplest way to use Control D is to configure your DNS settings manually. This method:
- Requires no software installation
- Works on any device that supports custom DNS
- Is perfect for static configurations

### Software is Available

For more advanced features and better user experience, you can use our software:
- Automatic configuration
- Dynamic IP support
- Advanced features like split tunneling
- Easy profile switching

## End User Mass Provisioning

For organizations deploying to multiple endpoints, we recommend using our [RMM integrations](/docs/rmm-integrations) or [API](/docs/api-reference).
