<header id="content-head"><div class="row clearfix"><div class="col-xs-9"># TTL Overrides

<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>Override default DNS record TTLs with your own.</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/ttl-overrides)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-is-dns-ttl&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-is-dns-ttl&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What is DNS TTL</div><Link aria-label=&quot;Skip link to What is DNS TTL&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-is-dns-ttl&quot;></Link>

<p>Every DNS record has a TTL, which instructs the OS to cache the DNS record for that period of time (in seconds) and not contact the DNS server for at least that long. This is used to reduce the amount of DNS queries clients make, and can have positive performance implications. </p>
<p>This Profile Option allows you to override the TTLs to whatever values you wish. This feature can sometimes be called &quot;Cache Boost&quot;, if you increase the TTLs to non-default values. By doing so, you instruct your OS to cache DNS records for longer periods of time. </p>
> ## <span class=&quot;callout-icon&quot;>ðŸš§</span><p>Know what you&#x27;re doing</p>

<p>Don&#x27;t change these randomly, or set them too high as this can have adverse consequences with CDNs. By high we mean &quot;measured in days&quot;.</p>

<p>There are 3 TTLs that you can tweak in Profile Options. </p>

## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;block-ttl&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-block-ttl&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Block TTL</div><Link aria-label=&quot;Skip link to Block TTL&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#block-ttl&quot;></Link>

<p>This overrides the TTL for DNS records that were blocked by Control D. The **default block TTL is 10s.** Increasing this value will result in fewer DNS lookups (for things that are blocked), but will also increase the time between you unblocking the domain and it actually becoming unblocked. </p>
## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;redirect-ttl&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-redirect-ttl&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Redirect TTL</div><Link aria-label=&quot;Skip link to Redirect TTL&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#redirect-ttl&quot;></Link>

<p>This overrides the default TTLs on DNS responses that were redirected by a Service, Custom Rule or Default Rule. The **default redirect TTL is 20s**. Increasing this value will result in fewer DNS lookups, but will also increase the time between you changing locations and your new location settings being actually enforced on a Device.</p>
## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;bypass-ttl&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-bypass-ttl&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Bypass TTL</div><Link aria-label=&quot;Skip link to Bypass TTL&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#bypass-ttl&quot;></Link>

<p>This overrides the TTL for all DNS records that were bypassed (not subject to any BLOCK or REDIRECT rule) and sent to the upstream resolver. Increasing this value will result in fewer DNS lookups, but can break websites if you set this too high.
