<header id="content-head"><div class="row clearfix"><div class="col-xs-9"># Connected Devices

<div class="excerpt"><div class="rm-Markdown markdown-body" data-testid="RDMD"><p>Physical devices that send DNS queries to Control D. You can think of them as seats.</p></div></div></div><div class="col-xs-3">[<i aria-hidden="true" class="icon icon-edit"></i> Suggest Edits](/edit/connected-devices)</div></div></header><div class="grid-container-fluid" id="content-container"><section class="content-body grid-75"><div dehydrated="# <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;what-are-connected-devices&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-what-are-connected-devices&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>What are Connected Devices</div><Link aria-label=&quot;Skip link to What are Connected Devices&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#what-are-connected-devices&quot;></Link>

<p>In normal SaaS terms, these are seats. Why didn&#x27;t we call them seats? Good question, but that&#x27;s a long story. </p>
<span aria-label=&quot;&quot; class=&quot;img lightbox closed&quot; role=&quot;button&quot; tabindex=&quot;0&quot;><span class=&quot;lightbox-inner&quot;><Image alt=&quot;&quot; loading=&quot;lazy&quot; src=&quot;https://files.readme.io/b9a750cf044efb6a0be38072f41493e5f511eda859dfdb21d609d259ee824a15-Connected_Devices_New_UI.png&quot; align=&quot;center&quot; class=&quot;&quot; width=&quot;55% &quot; caption=&quot;&quot; height=&quot;auto&quot; title=&quot;&quot;/></span></span>
<p>When you create a Control D **Endpoint** (used to be called a Device), it issues you a unique set of DNS resolvers, which you can configure to receive DNS queries from one or more physical devices used by humans. Some Endpoints are used by a single person, so feel free to put <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`1` into the box. </p>
<p>Other Endpoints like AD domain controllers, routers, or DNS servers are used by multiple physical devices (phone, laptop, etc), and our CFO said we need to make you pay for every physical device that ends up using Control D. So, if 100 devices are going to be sending their DNS queries to Control D, via this Endpoint, you need to put <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`100` into this box. </p>
## <div class=&quot;heading-anchor anchor waypoint&quot; id=&quot;honor-system&quot;></div><div class=&quot;heading-text&quot;><div id=&quot;section-honor-system&quot; class=&quot;heading-anchor_backwardsCompatibility&quot;></div>Honor System</div><Link aria-label=&quot;Skip link to Honor System&quot; class=&quot;heading-anchor-icon fa fa-anchor&quot; href=&quot;#honor-system&quot;></Link>

<p>We ask you to do your best to keep this count in sync with your actual usage. The amount of Connected Devices you select is not a hard limit, and nothing technically prevents you from putting <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`5` into the box, and using Control D on a network that has <button aria-label=&quot;Copy Code&quot; class=&quot;rdmd-code-copy fa&quot;></button>`5000` devices. However, we ask you to be fair and reasonable, since your honor is at stake. </p>
<p>We perform periodic audits, and check if everything looks reasonable given the average volume of DNS queries that a physical device emits on a daily basis. If you&#x27;re off by a little, we probably won&#x27;t care. But if you&#x27;re off by a few hundred of thousands, we&#x27;ll know and contact you. </p>
